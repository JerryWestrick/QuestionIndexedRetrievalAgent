# qira.prompt Template

Generated by `prompts/functions/qira --initialize`. Scans all corpus directories, reads each `corpus.md`, and composes this single prompt. Included in the system section of the prompt via `.include functions/qira.prompt`.

The template lives in the user's private `prompts/functions/` directory. The prompt engineer can modify it to suit their application — the template below is a reasonable default, not a mandate. The only hard requirement is that the generated prompt must document the markdown formats defined in the QI/RA interface spec, since those are the contract.

## What `--initialize` builds

1. **Corpus catalog table** — one row per corpus, from each `corpus.md`'s name and description. The corpus identifier (directory name) is the value the LLM uses in `qira_search(corpus=...)`.
2. **Worked example** — pulled from the first corpus's `corpus.md`.
3. **Everything else** — static text from the template. Tune it for your application.

## Template

The template uses Python `string.Template` substitution (`$variable`). This avoids conflicts with curly braces `{}` used in the markdown format documentation. `--initialize` calls `template.substitute(corpus_table=..., worked_example=...)`.

---

### Template file: `qira.prompt.template`

```
## QIRA Knowledge Retrieval

You have access to external knowledge through QIRA. Available corpora:

| Corpus | Description |
|--------|-------------|
$corpus_table

### Important

- **When a question is about topics covered by the corpora, answer from what you find there.** If you search and it's not covered, say so. If the question has nothing to do with the corpora, just answer normally.
- **Search and read until you have enough to fully answer.** Do not guess from partial results.
- **Reference section IDs in your answer** so the user can verify.

### Usage

When a question requires knowledge from a corpus:

1. **Search** — formulate a precise question requesting information you need, pick the corpus
   `qira_search(corpus="hr-regulations", question="How many vacation days per year?")`

2. **Browse** — scan the results, read matched questions and excerpts to judge relevance

3. **Read** — fetch full sections that look relevant
   `qira_read(section_id="hr-regulations:3.1.1")`

4. **Navigate** — follow cross-references, go to parent/sibling/child sections as needed

5. **Answer** — synthesize from what you read

### Worked Example

$worked_example

### Response Formats

**`qira_search` returns** per hit:
- `## -{distance}- {id} {title}` — match distance (lower = better), section ID (use with `qira_read`), title
- `> breadcrumb` — location in document tree
- `- *question*` — what this section answers
- excerpt text — preview of content
- If nothing matches: `## Nothing Found`

**`qira_read` returns:**
- `# {id} {title}` — section heading
- `> breadcrumb` — location in document tree
- full section content
- `## Subsections` — child sections you can navigate to

**Navigation from any ID** (e.g. `hr-regulations:3.2.4`):
- Parent: `hr-regulations:3.2`
- Sibling: `hr-regulations:3.2.5`
- Child: listed under `## Subsections`
```

### Variables

| Variable | Source | Example |
|----------|--------|---------|
| `$corpus_table` | One row per corpus, from each `corpus.md` | `| hr-regulations | Mexican labor law, employee benefits, company HR policies |` |
| `$worked_example` | From the first corpus's `corpus.md` | See example below |

### Example `$worked_example`

```
User asks: "As a part-time employee that started on June 1st, how many days vacation do I get this year?"

1. Do I know enough to answer? No — I need vacation rules for part-time employees.
   qira_search(corpus="hr-regulations", question="How many days vacation for part-time employees?")

2. Browse results. Read the most relevant hit.
   qira_read(section_id="hr-regulations:3.1.1")

3. Do I know enough to answer? Not yet — content mentions proration for mid-year starts but references a formula in another section.
   qira_read(section_id="hr-regulations:3.1.2")

4. Do I know enough to answer? Yes. Answer the user.
```
